# Strategic Planner

This skill creates comprehensive strategic plans for items in /Needs_Action, preparing them for human approval.

## Workflow

When this skill is invoked, follow these steps:

### 1. Run the Strategic Planner Script

Execute the strategic planning script:

```bash
python scripts/strategic_planner.py [vault_path]
```

- If `vault_path` is provided, process that vault location
- If omitted, process vault in current working directory

The script will:
- Scan `/Needs_Action` folder for items requiring planning
- Read and analyze each item's content and context
- Generate comprehensive `Plan.md` file with strategic recommendations
- Move the item folder (with Plan.md) to `/Pending_Approval`
- Update item status to 'pending_approval'

Each generated `Plan.md` follows this format:

```markdown

**Priority**: [High/Medium/Low]

[Clear statement of what needs to be accomplished]

[Background information and why this matters]

### Expected Outcome
[What success looks like]

---

## 2. Required MCP Tools

- **[Tool Name]**: [Purpose and usage]

## 3. Proposed Draft

### Approach
[Strategy and methodology]
```

### 3. Planning Process

For each item in /Needs_Action, the script:

**Step 1: Analyze Item**
- Read frontmatter metadata (type, priority, source)
- Extract key information (sender, subject, context)
- Understand the request or situation
- Identify urgency and importance

**Step 2: Define Objective**
- Clarify what needs to be accomplished
- Provide context and background
- Define expected outcomes
- Align with strategic goals

**Step 3: Identify MCP Tools**
- Determine which tools are needed (Gmail, LinkedIn, etc.)
- Specify primary vs. optional tools
- Note any configuration requirements
- Consider tool availability

**Step 4: Create Draft**
- Generate appropriate response or content
- Match tone to context and audience
- Include key messages and information
- Format for the target platform

**Step 5: Define Success**
- Set immediate success criteria
- Define long-term success metrics
- Make criteria measurable and specific
- Include tracking mechanisms

**Step 6: Assess Risks**
- Identify potential issues
- Propose mitigation strategies
- Create contingency plans
- Flag high-risk items

### 4. Item Organization

The script organizes items by creating folders:

```
Pending_Approval/
└── [YYYY-MM-DD]_[item-name]/
    ├── Plan.md (generated)
    └── [original-item].md (moved from Needs_Action)
```

Example:
```
Pending_Approval/
└── 2026-02-09_client-project-inquiry/
    ├── Plan.md
    └── 2026-02-09_gmail_client_project-inquiry.md
```

### 5. MCP Tool Identification

The script intelligently identifies required tools based on:

**Source Type:**
- `source: gmail` → Gmail MCP tool
- `source: linkedin` → LinkedIn MCP tool
- `type: email_task` → Gmail MCP tool
- `type: linkedin_*` → LinkedIn MCP tool

**Action Type:**
- Email response → Gmail MCP
- LinkedIn post/comment → LinkedIn MCP
- Document creation → File system tools
- Research → Web search tools

**Content Analysis:**
- Mentions of email → Gmail MCP
- Mentions of LinkedIn → LinkedIn MCP
- Mentions of attachments → File handling tools

### 6. Draft Generation

The script generates context-appropriate drafts:

**For Email Responses:**
```
Subject: Re: [Original Subject]

Dear [Name],

[Personalized greeting and acknowledgment]

[Main response content addressing their points]

[Call to action or next steps]

Best regards,
[Your Name]
```

**For LinkedIn Posts:**
```
[Engaging opening hook]

[Main content with value/insights]

[Call to action or question for engagement]

#hashtag1 #hashtag2
```

**For LinkedIn Comments:**
```
[Acknowledgment of their comment]

[Your perspective or additional value]

[Question or invitation to continue conversation]
```

### 7. Processing Rules

**Items to Process:**
- Must be in `/Needs_Action` folder
- Must have `status: pending` or `status: processing`
- Must be `.md` files
- Must not already have a Plan.md

**Items to Skip:**
- Already have `status: pending_approval`
- Already in `/Pending_Approval`
- Missing critical information
- Marked as `status: completed`

### 8. Status Updates

The script updates item status:

**Original item frontmatter:**
```yaml
status: pending_approval
planned_at: [timestamp]
plan_file: Plan.md
moved_to: /Pending_Approval/[folder-name]
```

## Integration with Workflow

This skill fits into the workflow:

1. **Gmail/LinkedIn Ingest** → Creates tasks in /Inbox or /Needs_Action
2. **Triage Inbox** → Processes and prioritizes items
3. **Strategic Planner** (this skill) → Creates plans in /Pending_Approval
4. **Human Approval Interface** → Reviews and approves plans
5. **Operational Executor** → Executes approved plans

## Company Handbook Integration

**If Company_Handbook.md exists:**
- Plans reference handbook guidelines
- Tone and style follow company standards
- Success criteria align with company goals
- Risk assessment considers company policies

**If Company_Handbook.md is not available:**
- Uses default professional standards
- Applies general best practices
- Recommends creating handbook for consistency

## Quality Standards

Each plan must meet these standards:

**Clarity:**
- Objective is specific and unambiguous
- Draft content is clear and well-structured
- Success criteria are measurable

**Completeness:**
- All required sections are filled
- Context is provided
- Risks are identified

**Appropriateness:**
- Tone matches the situation
- Content is professional
- Timing is considered

**Actionability:**
- MCP tools are correctly identified
- Draft is ready to execute
- Success criteria are trackable

## Error Handling

The script handles errors gracefully:
- **Missing information**: Creates plan with placeholders and flags for review
- **Unclear context**: Generates conservative plan with multiple options
- **Tool uncertainty**: Lists all potentially relevant tools
- **Complex items**: Creates detailed plan with step-by-step approach

All errors and warnings are logged for review.

## Important Notes

- Plans are drafts requiring human approval
- Script preserves original items
- Multiple items can be processed in batch
- Plans can be edited before approval
- High-priority items are flagged clearly

## Best Practices

1. **Review plans before approval** - Plans are starting points, not final
2. **Customize drafts** - Personalize generated content
3. **Verify tool availability** - Ensure MCP tools are configured
4. **Check timing** - Confirm execution timing is appropriate
5. **Consider context** - Plans should fit the broader strategy

Strategic planning is about thoughtful preparation, not just automation.
